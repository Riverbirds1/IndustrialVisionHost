## 5. 已发现并修复的问题

- 多线程同时打开相机可能重复启动采集循环。
- 多线程同时停止可能重复释放取消令牌和相机资源。
- 停止操作可能等待完整重连延迟，影响窗口关闭速度。
- OpenCV`Mat`在正常、调试图和异常路径下存在未释放风险。
- 模拟相机生成故障帧时，已分配的`Mat`在抛出异常前未释放。
- TCP断线、迟到响应和请求并发可能造成下一条报文错配。
- 数据库旧版本缺少新增字段，需要启动时兼容升级。

修复后，Debug和Release构建均为0警告、0错误，全套测试连续运行未出现失败。
