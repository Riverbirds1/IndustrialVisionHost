# 工业视觉上位机：从零到求职路线

## 最终目标

完成一套可演示、可测试、可扩展的工业视觉检测上位机，并能够在面试中独立解释：

- C#/.NET 基础、异步与多线程、资源管理和异常处理
- WPF、数据绑定、命令、MVVM、样式和线程模型
- 工业相机采集、图像处理、检测参数和结果可视化
- TCP/串口、Modbus TCP/RTU、PLC 握手及断线重连
- SQLite/SQL Server 数据存储、查询、追溯和报表
- 配方、权限、日志、报警、状态机和设备运行流程
- 单元测试、集成测试、性能诊断、发布部署和技术文档

## 学习方式

每个阶段固定完成六件事：

1. 理解核心概念和它解决的问题。
2. 完成一个独立小练习。
3. 将知识应用到当前项目。
4. 编译、运行并验证异常场景。
5. 回答本阶段面试题。
6. 更新项目文档和简历素材。

## 阶段路线

### 阶段 0：建立可靠基线（已完成）

- 模拟相机生成图像
- WPF 实时显示图像
- OpenCV 完成轮廓、面积和中心点检测
- 修复空值、资源泄漏、重复采集和线程退出问题
- 项目达到 0 警告、0 错误并能正常启动

验收：打开相机后可稳定显示；点击检测得到数量 1、面积约 19854、坐标约 (320,240)。

### 阶段 1：C# 与 WPF/MVVM 基础

- 复习类、接口、继承、泛型、委托、事件、LINQ
- 理解 `IDisposable`、异常、`Task`、`async/await` 和取消令牌
- 将窗口事件代码重构为 MVVM
- 使用属性绑定、命令和通知更新 UI
- 引入依赖注入，消除业务代码对具体相机类的直接依赖

验收：`MainWindow.xaml.cs` 只保留界面生命周期代码，按钮和状态全部通过绑定工作。

### 阶段 2：相机抽象与真实设备思维

- 定义 `ICamera` 接口和相机状态模型
- 实现模拟相机，可配置目标位置、大小、数量和故障
- 实现打开、关闭、采集、超时、断线和重连流程
- 理解软触发、硬触发、曝光、增益、帧率和图像缓冲
- 为以后接入海康/大华/OPT SDK 保留适配层

验收：不修改 ViewModel 即可切换不同相机实现；模拟断线后能够恢复。

### 阶段 3：可配置视觉检测流程（已完成）

- ROI、灰度化、滤波、阈值、形态学、轮廓筛选
- 面积、圆度、外接矩形和中心点
- 在原图叠加轮廓、中心、OK/NG 和耗时
- 参数校验、配方保存以及错误图保存
- 理解像素坐标与物理坐标、标定的基本概念

验收：可通过界面参数检测不同大小和位置的目标，并显示处理耗时与标注结果。

### 阶段 4：工业通信与 PLC 模拟（已完成）

- TCP/UDP、串口、RS232/RS485 基础
- Modbus TCP 与 Modbus RTU 数据模型
- 心跳、超时、重试、断线重连和报文日志
- 建立 PLC 模拟器，完成启动检测、忙、完成、OK/NG 握手

验收：无真实 PLC 时也能演示完整握手；拔掉连接后程序不崩溃并能恢复。

### 阶段 5：设备状态机与自动运行（已完成）

- Idle、Ready、Running、Completed、Fault、Emergency 等状态
- 手动/自动模式、启动、停止、复位和急停逻辑
- 防止重复指令、非法状态跳转和并发流程冲突

验收：通过状态图和自动化测试证明每个状态转换合法。

### 阶段 6：数据库与生产追溯（已完成）

- SQLite 和关系数据库基础
- EF Core、表设计、索引、事务和迁移
- 保存检测时间、批次、配方、结果、测量值和图片路径
- 条件查询、分页、统计、CSV/Excel 导出

验收：可按时间、批次和 OK/NG 查询历史记录，异常退出后数据保持完整。

### 阶段 7：工业软件常用模块（已完成）

- 参数配方管理
- 用户登录与操作权限
- 报警记录、确认与复位
- 操作日志、运行日志和通信日志
- 系统设置、语言与主题基础

验收：操作员、工程师、管理员拥有不同权限，关键参数修改可追溯。

### 阶段 8：稳定性、测试和性能（已完成）

- 单元测试、集成测试和设备模拟测试
- UI 与后台线程边界
- 内存泄漏、非托管资源、并发和死锁排查
- 长时间运行、异常注入和恢复测试
- 结构化日志与问题定位

验收：连续运行测试无明显内存增长；通信、相机和数据库故障都有明确反馈与恢复路径。

### 阶段 9：交付与求职材料

- Release 构建、配置文件、安装包和版本号
- README、架构图、流程图、通信协议和测试报告
- Git 提交历史与版本标签
- 演示视频、项目截图、简历项目描述和面试讲解

验收：一台干净的 Windows 电脑可按文档完成安装和演示；能够在 5 分钟内讲清项目价值、架构、难点和结果。

## 建议的加分项

完成主线后再选择：OPC UA、Siemens S7、三菱 MC/FINS、运动控制、MES/Web API、Halcon 或深度学习模型部署。不要在主线未稳定前同时堆叠这些技术。
